descripcion

la especificacion de css asume que el documenta se modela con arbol de elementos cada elemento contiene una lista ordenada de cero o mas elementos secundarios con una caena de texto antes de la lista,enmedio de los hijos despues de la lista,el unico elemento que no tiene padre se llama elemento raiz

css describe como debe posicionarse cada elemento y cada cadena de texto,transformando un arbol de documentos cajas rectangulares cuyo tama√±o o posiciones depende de sus propiedades existen cajas a nivel de bloques cajas a nivel de linea cajas a nivel de interno(inline)una caja a nivel de bloque es como un parrafo,una caja un a nivel de linea es como una linea de texto y las cajas nivel interno son como las palabras de una linea

las tres cajas se relacionan estrechamente como con el arbol de elementos. cada elemento se transforman en mas cajas si el elemento no es la raiz,entonces cada una de sus cajas es un hijo de una caja de elemento padre o un hijo en otras palabras puede haber mas o menos ajas que elementos pero cada caja pertenece a un elemento

las propiedades se establecen sobre los elementos y influyen en la forma en que cada elemento se convierte en cajas,pero en esta especificacion hay una intercambialidad de la propiedad P y la propiedad de una caja a menos que sea impornatnte distinguir entre la caja y el elemento ,por ejemplo cuando el elemento tiene varias cajas y no todas ellas tengan el mismo valor para la propiedad

las cajas tienen relleno,unborde de imagenes(ver las figuras)diferentes propiedades determinan el ancho(que podria ser cero) de cada uno de ellos

tipos de cajas

la propiedad display en conjunto con float positiondeterminan el tipo de caja que se genera para un elemento.los valores son los que siguen:

inline
cajas de linea
block
cajas de bloque
inline-block
una caja de bloque que es seguida por una sola caja de linea de manera similar a un elemento remplazado el interior de un bloque de linea se formatea como una caja negra y la caja por si misma es formateada como una caja en linea
list.item
una o mas cajas de bloque y una caja o mas de mercado,las cajas de mercado se definen en las listas de modulo
run-in
cajas de bloque o de linea,dependiendo el contexto.las propiedades se aplican a las cajas de ejecucion interna (run-in)en base su estado final
